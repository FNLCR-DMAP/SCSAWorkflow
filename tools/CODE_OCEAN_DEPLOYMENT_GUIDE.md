# SPAC Tools - Code Ocean Deployment Guide

## Overview

This guide explains how to deploy SPAC tools to Code Ocean platform using the synthesizer approach (v8.0).

**Key Features (v8.0):**
- **Named Parameters:** `--param_name=value` format (cleaner, more reliable)
- **Single `run.sh`:** Combined entry point + parameter parsing (no separate `main.sh`)
- **Automatic UI Generation:** `.codeocean/app-panel.json` auto-generates the App Panel

---

```
code_ocean_tools/                    # Output directory
├── boxplot/
│   ├── .codeocean/
│   │   └── app-panel.json          # UI config (auto-generates App Panel)

```

**Key Points:**
- Each tool has its own `format_values.py` in the same `code/` directory as `run.sh`
- 3 files per tool: `app-panel.json`, `run.sh`, `format_values.py`

## Step 3: Create Capsule in Code Ocean

### 3.1 Log in to Code Ocean
- Go to: https://codeocean.nih.gov (or your instance)

### 3.2 Create New Capsule from Git

1. Click **"+ New Capsule"**
2. Select **"Import from Git"** or **"Copy from Git"**
3. Enter your repository URL:
   - Separate repo: `https://github.com/fnlcr-dmap/spac-boxplot.git`
   - Monorepo: `https://github.com/FNLCR-DMAP/SCSAWorkflow.git`

### 3.3 Automatic App Panel Generation

**Key Point:** When Code Ocean detects `.codeocean/app-panel.json`, it automatically generates the App Panel UI!

You should see:
- Parameters organized by categories
- Default values populated
- Input types (text, dropdown, boolean) configured

### 3.4 Configure Environment

1. Go to **Environment** tab
2. Click **"Add Environment"**
3. Select **Docker** and enter: `nciccbr/spac:v2-dev`
4. Click **"Save"**

### 3.5 Attach Data

1. Go to **Data** tab
2. Click **"Add Data Asset"**
3. Select or upload your input data (`.pickle`, `.pkl`, or `.h5ad` files)

### 3.6 Test Run

1. Go to **App Panel** tab
2. Fill in parameter values
3. Click **"Reproducible Run"**

## File Details

### `.codeocean/app-panel.json`

This file defines the App Panel UI with **named parameters**:

```json
{
    "version": 1,
    "named_parameters": true,
    "general": {
        "title": "Boxplot [SPAC] [DMAP]",
        "instructions": "Create a boxplot visualization..."
    },
    "categories": [
        {"id": "figure_config", "name": "Figure Configuration", "help_text": "..."}
    ],
    "parameters": [
        {
            "id": "abc123",
            "category": "figure_config",
            "name": "Figure Width",
            "param_name": "Figure_Width",
            "type": "text",
            "value_type": "number",
            "default_value": 12
        }
    ]
}
```

**Key Setting:** `"named_parameters": true` enables `--param_name=value` argument format.

**Note:** For numeric parameters (`value_type: "number"`), `default_value` should be an actual number (e.g., `12` not `"12"`). For string parameters, use quoted strings.

**Parameter Type Mapping:**

| Blueprint Type | app-panel `type` | app-panel `value_type` |
|---------------|------------------|------------------------|
| STRING        | text             | string                 |
| INTEGER/INT   | text             | number                 |
| NUMBER/FLOAT  | text             | number                 |
| BOOLEAN       | list             | boolean                |
| SELECT        | list             | string (+ extra_data)  |
| LIST          | text             | string (comma-sep)     |

### `run.sh` (Single File - v8.0)

Combined entry point with all logic (no separate `main.sh`):

```bash
#!/usr/bin/env bash
set -euo pipefail

# Boxplot [SPAC] [DMAP]
# Generated by code_ocean_synthesizer.py v8.0 (Single run.sh)
# Named argument parsing: --param_name=value format

echo "=== Boxplot [SPAC] [DMAP] ==="

# Initialize parameters with default values
Figure_Width=12
Figure_Height=8
Horizontal_Plot="False"

# Parse named arguments dynamically (--param_name=value format)
for arg in "$@"; do
    case "$arg" in
        --*=*)
            key="${arg%%=*}"      # everything before '='
            value="${arg#*=}"     # everything after '='
            key="${key#--}"       # remove leading '--'
            export "$key=$value"
            ;;
    esac
done

# Find input data
INPUT=$(find -L ../data -type f \( -name "*.pickle" -o -name "*.pkl" -o -name "*.h5ad" \) 2>/dev/null | head -n 1)
if [ -z "$INPUT" ]; then echo "ERROR: No input file found in ../data"; exit 1; fi
echo "Input: $INPUT"

# Create output directories
mkdir -p /results/figures /results/jsons

# Create parameters JSON (numeric values WITHOUT quotes)
cat > /results/jsons/params.json << EOF
{
    "Upstream_Analysis": "$INPUT",
    "Figure_Width": $Figure_Width,
    "Figure_Height": $Figure_Height,
    "Horizontal_Plot": "$Horizontal_Plot"
}
EOF

# Normalize parameters
python ./format_values.py /results/jsons/params.json /results/jsons/cleaned_params.json \
    --bool-values Horizontal_Plot --list-values Feature_s_to_Plot

# Run SPAC template
python -c "from spac.templates.boxplot_template import run_from_json; run_from_json('/results/jsons/cleaned_params.json', output_dir='/results')"

echo "=== Completed. Outputs in /results/ ==="
```

**Key Features:**
- **Named argument parsing:** Handles `--param_name=value` format dynamically
- **Numeric values without quotes:** `"Figure_Width": $Figure_Width` (not `"$Figure_Width"`)
- **Output directory passed:** `output_dir='/results'` ensures correct output location

### `format_values.py`

**Location:** Each tool's `code/` directory (same directory as `run.sh`)

Normalizes parameters for templates:
- Converts `"True"` → `true` (Python boolean)
- Converts `"a, b, c"` → `["a", "b", "c"]` (list)
- Flattens nested section parameters
- Injects output configuration

Called from `run.sh` as `python ./format_values.py ...`

## Runtime Flow (v8.0 - Named Parameters)

```
┌─────────────────────────────────────────────────────────────────┐
│                    CODE OCEAN RUNTIME                           │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. User fills App Panel                                        │
│     ┌────────────────────────┐                                  │
│     │ Figure Width: [12]     │                                  │
│     │ Horizontal:   [False▼] │                                  │
│     └────────────────────────┘                                  │
│                   ↓                                             │
│  2. Code Ocean executes (named parameters):                     │
│     bash run.sh --Figure_Width=12 --Horizontal_Plot=False ...   │
│                   ↓                                             │
│  3. run.sh parses --key=value args → creates params.json        │
│                   ↓                                             │
│  4. format_values.py normalizes → cleaned_params.json           │
│                   ↓                                             │
│  5. SPAC template runs with cleaned params                      │
│                   ↓                                             │
│  6. Results saved to /results/                                  │
│     ├── figures/                                                │
│     ├── jsons/params.json (debug)                               │
│     └── dataframe.csv                                           │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**Key Difference from v7.x:** Named parameters (`--param=value`) instead of positional args (`$1, $2, ...`)

## Batch Deployment (40 Tools)

### Generate all capsules:
```bash
python code_ocean_synthesizer.py blueprint_jsons/ -o code_ocean_tools/
```

### Create repos and push (script):
```bash
#!/bin/bash
for tool_dir in code_ocean_tools/*/; do
    tool_name=$(basename "$tool_dir")
    echo "Deploying: $tool_name"
    
    cd "$tool_dir"
    git init
    git add .
    git commit -m "Initial $tool_name capsule"
    
    # Create GitHub repo (requires gh CLI)
    gh repo create "fnlcr-dmap/spac-$tool_name" --public --source=. --push
    
    cd ../..
done
```

## Troubleshooting

### App Panel not appearing
- Verify `.codeocean/app-panel.json` exists
- Check JSON syntax: `python -m json.tool .codeocean/app-panel.json`
- Ensure `"version": 1` is present
- Ensure `"named_parameters": true` is set

### "format_values.py not found"
- Ensure `format_values.py` is in the same directory as `run.sh`
- Check that `run.sh` calls it correctly: `python ./format_values.py ...`

### Parameters not being parsed
- v8.0 uses named parameters: `--param_name=value` format
- Check `run.sh` has the dynamic argument parsing loop
- Enable debug output by checking "Arguments received:" in run output

### "No input file found"
- Attach data asset in Code Ocean Data tab
- Verify file extension is `.pickle`, `.pkl`, or `.h5ad`

### Template not found
- Ensure Docker image `nciccbr/spac:v2-dev` contains `spac.templates`
- Check template name matches: `{tool_name}_template`

### Numeric values appearing as strings
- Check `run.sh` writes numeric params WITHOUT quotes in JSON
- Correct: `"Figure_Width": $Figure_Width`
- Wrong: `"Figure_Width": "$Figure_Width"`

### Results not appearing
- All outputs must go to `/results/` directory
- Check `run.sh` creates `/results/figures` and `/results/jsons`
- Verify template call includes `output_dir='/results'`

### Key v8.0 Changes
| Feature | v7.x | v8.0 |
|---------|------|------|
| Files per tool | 3 (app-panel.json, run.sh, main.sh) | 3 (app-panel.json, run.sh, format_values.py) |
| Parameter format | Positional ($1, $2, ...) | Named (--param=value) |
| app-panel.json | `named_parameters: false` | `named_parameters: true` |
| format_values.py | Shared at root | Per-tool (same dir as run.sh) |

### Key Conor Insight
> "If you create a capsule by importing a `.codeocean/app-panel.json` from a git repo, 
> the corresponding capsule app panel will be generated."

> "run.sh is the only requirement to run a capsule"

This eliminates manual App Panel building and simplifies the structure!
